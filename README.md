# ISL to Speech

Real-time Indian Sign Language (ISL) fingerspelling → text → speech.
Runs entirely in the browser. No server. No data leaves the device.

---

## Architecture

```
Webcam → MediaPipe HandLandmarker (GPU) → Hand ROI crop
  → ONNX Runtime Web (WASM) → Your classifier
  → Hold-confirm buffer → Web Speech API → Audio output
```

**Zero-backend stack:**

| Layer | Technology |
|---|---|
| Hand detection | MediaPipe Tasks Vision (CDN) |
| Inference | ONNX Runtime Web 1.20 (WASM) |
| Speech | Web Speech API (browser-native) |
| Hosting | Vercel (static) |

---

## Prerequisites

- Python ≥ 3.8 (local model verification only)
- A trained ISL classifier exported to `.onnx` format
- Node.js + Vercel CLI (for deployment)

---

## Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Export your model to ONNX

```python
# From Ultralytics YOLO
model.export(format='onnx', imgsz=224, simplify=True)
# Output: best.onnx
```

### 3. Verify the model and generate labels.json

```bash
python test_onnx.py --model best.onnx
# Optional: test on a real image
python test_onnx.py --model best.onnx --image hand.jpg
```

This writes `labels.json` automatically. Copy it to the project root.

### 4. Place your model files in the project root

```
isl-to-speech/
├── best.onnx       ← your exported model
├── labels.json     ← generated by test_onnx.py
```

## Deployment (Vercel)

```bash
npm install -g vercel
cd isl-to-speech
vercel --prod
```

`vercel.json` is pre-configured with the required COEP/COOP headers for
ONNX Runtime Web's WASM threading (`SharedArrayBuffer`).

## Configuration

Edit `window.ISL_CONFIG` in `index.html` to tune behaviour:

| Key | Default | Description |
|---|---|---|
| `imgsz` | `224` | Input resolution expected by your model |
| `confThreshold` | `0.60` | Minimum softmax confidence to accept a letter |
| `marginThreshold` | `0.20` | Minimum gap between top-2 class scores |
| `holdFrames` | `7` | Consecutive frames a letter must hold before committing |
| `defaultTolerance` | `2` | Frames of no-hand allowed before resetting hold |
| `speakTrigger` | `12` | Frames of no-hand before auto-speaking the buffer |
| `ttsLang` | `en-US` | BCP-47 language tag for Web Speech API |
| `ttsRate` | `0.95` | Speech rate (0.1 – 10) |
| `executionProvider` | `wasm` | `wasm` (safe) or `webgl` (faster on GPU devices) |

## Troubleshooting

**"ONNX Runtime Web not loaded"**
The ORT `<script>` tag must be synchronous (no `defer`/`async`). Check `index.html`.

**"Camera access denied"**
Vercel serves over HTTPS automatically. For local dev, use `localhost` not an IP.

**Model loads but all predictions are `no_gesture`**
Lower `confThreshold` to `0.45` in `ISL_CONFIG` and retest.

**Labels are wrong / off by one**
Re-run `test_onnx.py` — it reads class order directly from ONNX metadata.
